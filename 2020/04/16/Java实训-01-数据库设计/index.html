数据库设计

  • 1.提取数据实体

    • 用户
    • 音乐
    • 列表
    • 艺术家
    • 专辑

  • 2.提取数据项

    • 用户（帐号、密码、昵称、性别，头像、年龄、省、市、区、邮箱、手机号、个性签名）
    • 音乐（id，name，duration，copyrightId，status，alias，rtype，ftype，mvid，fee，rUrl，mark）
    • 列表（列表名称、创建日期、列表描述）
    • 艺术家（id,name,picUrl,alias,albumSize,picId,img1v1Url,img1v1,trans）
    • 专辑（id,name,publishTime,size,copyrightId,status,picId,mark）

  • 3.通过数据库设计范式约束数据项的提取

    • 数据表中的字段不可再分
    • 不存在非关键字段对关键字段的部分函数依赖
    • 不存在非关键字段之间的传递依赖

  • 4.绘制E-R图

  • 5.三线图

    • 表名:tb_user

      字段名 类型/长度 约束 字段说明
      id int 主键，自动增长 用户ID
      username varchar(30) NOT NULL, UNIQUE 用户登录账号
      password varchar(16) NOT NULL 用户登录密码
      nickname varchar(30) 用户昵称
      gender char(2) 用户性别
      pic_url varchar(200) 用户头像路径
      age int 用户年龄
      province varchar(20) 省份
      city varchar(40) 城市
      area varchar(40) 区
      email varchar(100) 邮箱
      phone char(11) UNIQUE 电话
      remark varchar(400) 个性签名
